ConfigMap - Red Hat Fuse

This booster provides a basic example of how to use a ConfigMap to externalize configuration for an Apache Camel route. A ConfigMap is an object that you can use to inject configuration data (in simple key and value pair format) into one or more Linux containers.

This page invokes the Greeting Service every 5 seconds. The Greetings Service responds by using the booster.nameToGreet value from the fuse-configmap-booster ConfigMap.

To demonstrate how the ConfigMap works:

  • Modify the booster.nameToGreet value by using the OpenShift console or CLI.
  • View the changed value in the Greeting Service's results window.
